## 1. Опис пристрою

ESP32 Метеостанція — це автономний пристрій для вимірювання та відображення метеорологічних параметрів у реальному часі. Пристрій:

- вимірює **температуру**, **вологість** та **атмосферний тиск** за допомогою датчика BME280;
- відображає дані на кольоровому **TFT-дисплеї** (320×240 пікселів);
- забезпечує доступ до даних через **веб-браузер** у локальній мережі;
- зберігає **архів вимірювань** кожні 5 хвилин у форматі CSV;
- синхронізує точний час через **NTP-сервер** (або з апаратного годинника RTC DS3231);
- підтримує **статичну IP-адресу** для стабільного доступу.

---

## 2. Технічні характеристики

| Параметр | Значення |
|---|---|
| Мікроконтролер | ESP32 DevKit V1 |
| Датчик | BME280 (I²C) |
| Дисплей | TFT ILI9341, 320×240 пікселів |
| Годинник реального часу | DS3231 (I²C) + батарейка CR2032 |
| Живлення | 5V USB (мінімум 1А) |
| Протокол зв'язку | Wi-Fi 802.11 b/g/n (2.4 ГГц) |
| Файлова система | SPIFFS (вбудована Flash-пам'ять) |
| Веб-сервер | AsyncWebServer (порт 80) |
| IP за замовчуванням | **192.168.1.230** (статична) |
| Часовий пояс за замовч. | EET/EEST (UTC+2, Київ) |

## 3. Перше увімкнення та підключення до Wi-Fi

### Крок 1 — Подача живлення

Підключіть пристрій через **Micro-USB** до блока живлення ≥ 1А. На екрані з'явиться початковий екран ініціалізації.

> ⚠️ **Важливо:** Використовуйте якісне джерело живлення. Просадки напруги через слабкий блок живлення є найпоширенішою причиною нестабільної роботи Wi-Fi.

### Крок 2 — Спроба підключення до Wi-Fi

Пристрій спробує підключитися до **збереженої Wi-Fi мережі**. На екрані відобразиться повідомлення про спробу підключення.

**Якщо підключення успішне:**
- На екрані з'явиться призначена IP-адреса
- Час синхронізується з NTP-сервером автоматично
- Станція перейде в нормальний режим роботи

### Крок 3 — Режим Точки Доступу (AP Mode)

Якщо збереженої мережі немає або вона недоступна, пристрій автоматично перейде в **режим AP (Access Point)**:

1. На вашому смартфоні або ПК знайдіть Wi-Fi мережу з назвою **`ESP32_Config`**
2. Пароль для підключення: **`12345678`**
3. Відкрийте браузер та перейдіть за адресою: **`http://192.168.4.1`**
4. У веб-формі введіть назву (SSID) та пароль вашої домашньої Wi-Fi мережі
5. Натисніть **«Зберегти»** — пристрій перезавантажиться та підключиться до мережі

## 4. Веб-інтерфейс

Після підключення до Wi-Fi відкрийте браузер та введіть IP-адресу пристрою (за замовчуванням **`http://192.168.1.230`**).

### 4.1 Головна сторінка (`/`)

Відображає **поточні дані** з датчика BME280:

| Показник | Одиниці виміру |
|---|---|
| Температура | °C (з урахуванням калібрування) |
| Вологість | % |
| Тиск | Па (Паскалі) |
| Тиск | мм рт. ст. |

На сторінці також доступні **інтерактивні графіки** зміни температури, вологості та тиску за поточну добу.

### 4.2 Архів даних (`/logs`)

Відображає список доступних CSV-файлів архівів (один файл = один день). Для кожного файлу доступні:
- перегляд у вигляді **таблиці з графіком** (`/log_viewer?date=YYYY-MM-DD`)
- завантаження у форматі **CSV** (`/log/YYYY-MM-DD.csv`)

### 4.3 Ручне збереження (`/save_now`)

Примусово зберігає поточний буфер вимірювань в CSV-архів без очищення масиву. Зручно для перевірки або ручного резервного копіювання.

### 4.4 Налаштування (`/settings`)

Сторінка конфігурації пристрою (детальніше — у розділі [5. Налаштування](#5-налаштування)).

### 4.5 Перезавантаження

Кнопка **«🔁 Перезавантажити ESP32»** на сторінці налаштувань негайно перезапускає пристрій. Після перезавантаження (≈ 8 секунд) браузер автоматично переходить на головну сторінку.

## 5. Налаштування

Перейдіть на **`http://192.168.1.230/settings`**

### 5.1 Загальні налаштування

| Поле | Опис |
|---|---|
| **Назва пристрою** | Відображається в заголовку веб-інтерфейсу |
| **WiFi SSID** | Назва вашої домашньої Wi-Fi мережі |
| **Пароль WiFi** | Пароль від Wi-Fi (є кнопка «Показати пароль») |

### 5.2 Часовий пояс

| Поле | Опис |
|---|---|
| **UTC Зміщення** | Введіть `2` для Києва (UTC+2 взимку) |
| **Літній час (DST)** | Поставте ✓, щоб автоматично переходити на літній час (+1 год у березні–жовтні) |

> 📝 **Приклад для України:** UTC Зміщення = `2`, DST = ✓

### 5.3 Калібрування датчиків

Датчик BME280, розташований близько до нагрітих компонентів, може показувати завищену температуру.

| Поле | Опис | Рекомендований діапазон |
|---|---|---|
| **Калібрування температури** | Від'ємне значення знижує показання (наприклад `-2.5`) | від -10 до +10 °C |
| **Калібрування вологості** | Корекція показань вологості | від -50 до +50 % |

> 💡 **Порада:** Порівняйте показання з еталонним термометром і введіть різницю зі зворотнім знаком.

### 5.4 Модулі (екранна карусель)

Увімкніть або вимкніть окремі слайди карусельного відображення на TFT-дисплеї:

| Перемикач | Що відображається |
|---|---|
| **Годинник** | Поточний час, дата, день тижня |
| **Барометр** | Температура, вологість, тиск |
| **Заповіт** | Вірш Т. Г. Шевченка |
| **Фото** | Фотографія (персональний слайд) |
| **Слава** | «Слава Україні!» |
| **Тризуб** | Графічний символ тризуба |
| **Москалі їбучі** | Патріотичний слайд |

> ⚙️ Після натискання **«💾 Зберегти»** налаштування записуються в енергонезалежну пам'ять (NVS) і зберігаються навіть після відключення живлення.

## 6. Екранна індикація (TFT)

Дисплей працює в автоматичному режимі **«Карусель»** — послідовно змінює активовані слайди.

### Тривалість слайдів

| Режим | Тривалість |
|---|---|
| Годинник (дата/час) | ~10 секунд |
| Барометр (дані BME280) | ~10 секунд |
| Графічні заставки | ~10 секунд |
| Цитата Шевченка | ~15 секунд |

### Режим без активних модулів

Якщо всі перемикачі вимкнені, екран відображає **стандартний фон** (темний екран без вмісту).

### Оновлення даних на екрані

- Дані з датчика **оновлюються кожні 30 секунд**
- Архівна точка записується **кожні 5 хвилин** (усереднена за 30-секундними вимірами)

## 7. Архівування даних

### Логіка збереження

| Подія | Частота |
|---|---|
| Зчитування датчика | Кожні 30 секунд |
| Запис в буфер пам'яті | Кожні 5 хвилин (усереднене значення) |
| Збереження буфера в CSV | 6 разів на добу (00:00, 04:00, 08:00, 12:00, 16:00, 20:00) |
| Очищення старих архівів | Автоматично — видаляються файли старші **14 днів** |

### Формат CSV-файлу

```
Time,Temperature,Humidity,Pressure
2026-04-25 08:00,21.3,58.2,99850.1
2026-04-25 08:05,21.4,57.9,99847.3
...
```

### Де зберігаються файли

Файли зберігаються у файловій системі SPIFFS за шляхом `/log/YYYY-MM-DD.csv`.  
Загальний обсяг SPIFFS обмежений (~1.5 МБ для файлів). Автоочищення за 14 днів гарантує, що пам'ять не переповниться.

> ⚠️ **Увага:** Не відключайте живлення під час архівування (≈ 10–30 секунд після 00:00, 04:00, 08:00 і т.д.). Це може призвести до пошкодження файлу архіву за поточну добу.

## 8. Вирішення проблем

### 🔴 Пристрій не з'являється в мережі / не відповідає на пінг

1. Перевірте чи правильна IP-адреса — за замовчуванням **`192.168.1.230`**
2. Відкрийте Serial Monitor в PlatformIO (швидкість **115200 бод**) — там буде вказана реальна отримана IP
3. Переконайтесь, що роутер і пристрій знаходяться в одній підмережі (`192.168.1.x`)
4. Якщо IP не `192.168.1.230` — змініть статичну адресу в коді або налаштуйте резервацію на роутері

### 🔴 Некоректний час на екрані (відстає або поспішає)

1. Перевірте значення **UTC Зміщення** в налаштуваннях (для Києва: `2`)
2. Переконайтесь що стоїть ✓ на **Літній час (DST)** якщо зараз літо
3. Якщо NTP недоступний — час береться з RTC DS3231. Перевірте **батарейку CR2032** в модулі годинника
4. Після зміни налаштувань часового поясу натисніть **«Зберегти»** та **перезавантажте** пристрій

### 🔴 Кнопка «Перезавантажити» не реагує

1. Переконайтесь що завантажили **Filesystem Image** (SPIFFS) через PlatformIO після оновлення `settings.html`
2. Після завантаження SPIFFS обов'язково завантажте і **прошивку** (Upload)
3. Перевірте в Serial Monitor — при натисканні має з'явитись повідомлення про рестарт

### 🔴 Пристрій сам перезавантажується / зависає

Найпоширеніші причини:
- **Слабке живлення** — замініть блок живлення на ≥ 1А
- **Фрагментація пам'яті** — через довгу роботу; натисніть кнопку рестарту раз на кілька тижнів
- **Wi-Fi нестабільний** — перевірте рівень сигналу, наблизьте пристрій до роутера

### 🔴 Відображаються некоректні дані температури (завищена)

- Введіть від'ємне **Калібрування температури** в налаштуваннях (наприклад `-2.5`)
- Переконайтесь що корпус пристрою добре вентилюється

### 🔴 Графік порожній або не завантажується

- Дані з'являються в буфері тільки після першого 5-хвилинного циклу
- Після рестарту — зачекайте 5–10 хвилин
- Якщо архіви за минулі дні порожні — перевірте що годинник показує правильну дату

### 🔴 Сторінки завантажуються дуже повільно

- Це нормально при відкритті великих CSV-архівів (1000+ рядків)
- Спробуйте переглядати архів з обмеженням: `/bme_chart_data_archive?date=YYYY-MM-DD&limit=200`

## 9. Оновлення прошивки

### Необхідні інструменти

- [VS Code](https://code.visualstudio.com/) + розширення [PlatformIO IDE](https://platformio.org/)
- Репозиторій: `git clone https://github.com/Amator57/esp-32_weather.git`

### Порядок завантаження

> ⚠️ **Важливо:** Завжди завантажуйте SPIFFS **перед** прошивкою, якщо змінювались файли в папці `data/`.

**1. Завантаження файлової системи (SPIFFS):**
```
PlatformIO → PROJECT TASKS → esp32dev → Platform → Upload Filesystem Image
```

**2. Завантаження прошивки:**
```
PlatformIO → PROJECT TASKS → esp32dev → General → Upload
```

**3. Моніторинг:**
```
PlatformIO → PROJECT TASKS → esp32dev → General → Monitor  (115200 baud)
```

### Що не потребує повторного завантаження SPIFFS

Якщо змінювалися **тільки** файли `.cpp` або `.h` (код без змін в папці `data/`) — достатньо завантажити лише прошивку (крок 2).
